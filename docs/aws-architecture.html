GovCloud Migration Checklist — What Changes, What Doesn't

  • Same Docker containers, same application code — zero changes to app logic
  • RDS: pg_dump from us-east-1 → pg_restore into us-gov-east-1 RDS instance
  • Entra ID: swap personal tenant config for agency tenant (environment variable)
  • OpenAI → self-hosted Llama 3.1 on a GPU instance (single env var swap in model router)
  • NAT instance → managed NAT Gateway (GovCloud compliance preference)
  • Add CloudTrail, Config, GuardDuty for FedRAMP Moderate controls
  • Add VPC Flow Logs and centralized logging (CloudWatch → S3)
  • TLS cert: ACM in us-east-1 → ACM in us-gov-east-1 (same process)
  • IAM roles: recreate in GovCloud account (same policies, different ARNs)
